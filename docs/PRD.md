# 가족 독서 기록 및 커뮤니티 플랫폼 - PRD (Product Requirements Document)

## 1. 서비스 개요

### 1.1 개요
**woojuLib**는 가족 구성원들이 함께 독서 기록을 관리하고, 독서 경험을 공유하며, AI 기반 추천을 받는 통합 플랫폼입니다. 가족만의 프라이빗한 커뮤니티 공간에서 각자의 독서 여정을 기록하고 함께 성장할 수 있습니다.

### 1.2 서비스의 가치 제안
- **가족 맞춤형**: 개인별 독서 습관과 선호도를 학습하여 맞춤형 도서 추천
- **함께 읽는 경험**: 가족 구성원 간 독서 감상 공유로 소통 기회 제공
- **시각적 성과 추적**: 개인 및 가족 전체의 독서 현황을 한눈에 파악
- **프라이빗한 커뮤니티**: 가족 멤버에게만 열린 폐쇄형 커뮤니티 공간

### 1.3 목표
- **월별 가족 독서량 증대**: 월별 가족 전체 독서 권수 목표 설정 및 달성
- **독서 문화 형성**: 가족 내 적극적인 독서 토론 및 추천 문화 조성
- **개인별 독서 성장**: 각 가족 구성원의 선호도 파악 및 맞춤형 콘텐츠 제공

## 2. 타겟 유저 및 페르소나

### 2.1 가족 구성원별 페르소나

#### 페르소나 1: "러더스 맘" - 샛별 (1986년생, 40세, 엄마)
**특성**
- 역할: 가족의 독서 활동 주도자 및 독서 문화 기획자
- 독서량: 월 4~5권
- 선호 장르: 소설, 자기계발, 심리학, 에세이
- 기술 능숙도: 중상 (스마트폰 사용 능숙, SNS 활동 중)

**주요 니즈**
- 세 자녀의 독서 습관을 체계적으로 관리하고 격려하고 싶음
- 각 아이의 성장 단계에 맞는 도서를 찾아 추천하고 싶음
- 가족 전체의 독서 현황을 한눈에 파악하고 싶음
- 자신도 계속 읽으면서 아이들과 함께 성장하고 싶음

**주요 행동**
- 매주 새로운 도서 추천 리스트를 확인하고 아이들에게 제안
- 아이들의 리뷰에 댓글로 응원과 질문을 남김
- 월간 통계를 보면서 가족의 읽기 목표를 점검하고 동기부여
- 자신의 독서 감상을 상세히 기록하고 아이들의 반응 기다림

---

#### 페르소나 2: "초딩 독서광" - 우주 (2018년생, 8세)
**특성**
- 역할: 독서에 진심인 활발한 참여자
- 독서량: 월 6~8권 (많이 읽음)
- 선호 장르: 판타지, 모험, 과학소설, 추리소설
- 기술 능숙도: 높음 (게임이나 앱 사용에 익숙)

**주요 니즈**
- 자신이 좋아하는 책 말고도 새로운 장르의 재미있는 책을 발견하고 싶음
- 엄마나 형/언니와 자신이 읽은 책에 대해 이야기하고 싶음
- 읽은 책들을 자랑스럽게 기록해서 보여주고 싶음
- 혼자가 아닌 가족이 함께 책을 읽는 경험을 원함

**주요 행동**
- 자주 앱에 접속하여 읽은 책을 빠르게 기록하고 재미있었다는 평점 남김
- 형/언니나 엄마가 남긴 다른 책의 리뷰를 읽고 "나도 읽고 싶어!" 댓글 작성
- 엄마의 추천을 적극 수용하고 읽은 후 자신의 생각 공유
- 월간 독서 현황을 자랑스럽게 본인이 확인하고 통계 챙김

---

#### 페르소나 3: "신입 독서자" - 유주 (2019년생, 7세)
**특성**
- 역할: 독서 활동을 시작한 새로운 참여자
- 독서량: 월 1~2권 (정기적으로 늘어나는 중)
- 선호 장르: 청소년 소설, 로맨스, 일상 에세이, 감성 도서
- 기술 능숙도: 매우 높음 (SNS 활용 능숙, 온라인 문화에 익숙)

**주요 니즈**
- 또래 청소년들이 읽는 책을 알고 싶음
- 독서에 빠져 있는 엄마와 우주의 열정을 조금씩 이해하고 싶음
- 나이에 맞는 책을 찾되, 너무 쉽지 않은 책을 읽고 싶음
- SNS처럼 편하게 자신의 책 감상을 남기고 싶음

**주요 행동**
- 엄마나 우주가 추천한 책을 한 권씩 읽으며 시험해봄
- 읽은 책의 감상을 짧고 간단하게 남김 (한두 문장)
- 엄마나 우주의 댓글과 반응을 통해 독서 동기 부여받음
- 자신이 관심 가는 책을 엄마나 우주에게 물어보거나 추천해달라고 요청

---

#### 페르소나 4: "독서 편식자" - 호섭 (1984년생, 42세, 아빠)
**특성**
- 역할: 특정 장르에만 깊이 있는 선택적 독자
- 독서량: 월 2~3권 (정해진 장르만)
- 선호 장르: 경제, 비즈니스, 역사 (거의 이 세 장르만 읽음)
- 기술 능숙도: 낮음 (스마트폰 사용 서툼, 간단한 기능만 사용)

**주요 니즈**
- 자신이 선호하는 분야의 책을 놓치지 않고 싶음
- 가족이 읽고 있는 책들을 가끔 들여다보면서 가족과 연결되고 싶음
- 복잡하지 않은 인터페이스에서 편하게 기록하고 싶음
- 자신의 독서 취향이 존중받기를 원함

**주요 행동**
- 월에 한두 권씩 자신이 원하는 분야의 책을 찾아 추가
- 읽은 책에 대해 짧은 코멘트와 평점만 남김 (상세한 리뷰는 하지 않음)
- 가끔 우주나 유주의 리뷰를 훑어보면서 "저런 책도 있네" 정도로 반응
- 엄마가 권하는 책에는 대부분 "알겠어" 하면서도 자신이 선택한 책을 우선시 함
- 앱 사용에 어려움이 있을 때 엄마가 대신 도와줌

## 3. 기능 요구사항 정의

### 3.1 사용자 관리 (User Management)

#### 3.1.1 가족 계정 생성 및 관리
- **가족 그룹 생성**: 초대자(부모 역할)가 가족 그룹을 생성
- **가족 구성원 초대**: QR 코드 또는 초대 링크를 통한 가족 추가
- **프로필 설정**: 각 멤버별 프로필 사진, 닉네임, 소개글 설정
- **역할 설정**: 부모/자식/기타 역할 설정 (콘텐츠 관리 권한 상이)

#### 3.1.2 개인 프로필
- **독서 습관 설정**: 선호 장르 (다중선택), 월 목표 권수 설정
- **개인 대시보드**: 나의 독서 현황, 최근 활동, 통계 요약

#### 3.1.3 설정 페이지 (Settings Page)
사용자가 자신의 프로필 정보와 독서 습관을 언제든지 수정할 수 있는 설정 페이지를 제공합니다.

**3.1.3.1 프로필 설정**
- **기본 정보 수정**: 닉네임, 프로필 사진, 소개글 수정
- **개인 정보 수정**: 이메일, 비밀번호 변경
- **계정 삭제**: 가족에서 탈퇴 또는 계정 완전 삭제

**3.1.3.2 독서 습관 설정**
- **선호 장르 수정**: 언제든지 선호 장르를 추가/제거
  - 선택 가능 장르: 소설, 자기계발, 에세이, 판타지, 모험, 추리, SF, 로맨스, 경제, 비즈니스, 역사, 인문학, 청소년 도서, 어린이 도서, 고전, 웹소설, 무협 등
- **월 독서 목표 설정**: 월별 읽고 싶은 책의 권수 설정 (기본값: 1~5권)
- **독서 시간 목표**: 일일 또는 주간 목표 시간 설정 (선택사항)

**3.1.3.3 알림 설정**
- **활동 알림 여부**: 가족이 새 책을 다 읽었을 때 알림 받을지 여부
- **댓글 알림 여부**: 자신의 리뷰에 댓글이 달렸을 때 알림 받을지 여부
- **추천 알림 여부**: 새로운 도서 추천이 생겼을 때 알림 받을지 여부
- **목표 달성 알림**: 월간 목표 달성 축하 알림 받을지 여부

**3.1.3.4 개인정보 & 프라이버시 설정**
- **가족 내 표시 여부**: 내 리뷰와 활동을 가족에게 공개할지 설정
- **통계 포함 여부**: 가족 통계에 내 데이터 포함할지 설정

**3.1.3.5 UI 설정** (개인화)
- **다크 모드**: 다크/라이트 모드 선택
- **언어 설정**: 한국어, 영어 등 선택
- **폰트 크기**: 접근성을 위한 폰트 크기 조절 (할아버지처럼 낮은 기술 능숙도를 위함)

**3.1.3.6 연결된 계정**
- **SNS 연동 관리**: Google, Apple, Naver, Kakao 계정 연동 상태 확인 및 해제

#### 3.1.4 인증 및 보안
- 이메일/비밀번호 기반 회원가입
- SNS 연동 로그인 (Google, Apple, Naver, Kakao)
- 가족 내 프라이버시 보호 (외부 노출 불가)

---

### 3.2 독서 기록 관리 (Reading Log Management)

#### 3.2.1 도서 검색 및 추가
- **도서 DB 연동**: Google Books API, Naver Books API, Kakao Books API 연동
- **상세 검색**: 제목, 저자, ISBN, 카테고리별 검색 지원
- **도서 정보 표시**: 표지 이미지, 제목, 저자, 출판사, 출판일, 요약, 별점 등
- **도서 추가**: 읽기 시작하는 책을 "읽고 있는 책" 목록에 추가

#### 3.2.2 읽기 진행 관리
- **읽기 상태 추적**: 읽기 전 → 읽는 중 → 읽음 (3단계)
- **페이지 추적** (선택사항): 현재 읽은 페이지 기록
- **읽기 시간 기록**: 읽은 날짜, 소요 시간 기록
- **일일 목표 설정**: 일일 읽기 페이지 또는 시간 목표 설정

#### 3.2.3 독서 기록 관리
- **읽은 책 목록**: 과거에 읽은 책들을 시간순, 장르순으로 정렬
- **개인 서재**: 현재 읽는 책, 읽고 싶은 책, 읽은 책 분류
- **삭제 및 수정**: 잘못 추가된 도서 수정/삭제 가능

---

### 3.3 리뷰 및 감상 공유 (Review & Sharing)

#### 3.3.1 독서 리뷰 작성
- **리뷰 작성**: 책을 다 읽은 후 생각과 감상 기록
- **평점 부여**: 1~5점 별점 평가
- **감상평 작성**: 자유로운 형식의 감상평, 인상깊은 구절 기록
- **스포일러 태그**: 리뷰에 스포일러 경고 표시

#### 3.3.2 가족 간 상호작용
- **리뷰 조회**: 다른 가족 구성원들의 리뷰와 평점 확인
- **댓글 기능**: 리뷰에 대한 댓글로 의견 나누기
- **좋아요/하트**: 인상깊은 리뷰에 반응하기
- **알림**: 내 리뷰에 댓글이 달렸을 때 알림

#### 3.3.3 가족 피드 (Timeline)
- **통합 피드**: 가족 전체의 리뷰, 댓글 활동 보기
- **필터링**: 특정 멤버의 활동만 보기, 특정 책에 대한 의견만 보기
- **시간순 정렬**: 최신 활동부터 표시

---

### 3.4 AI 기반 도서 추천 (AI-Powered Recommendations)

#### 3.4.1 개인맞춤 추천
- **독서 기반 추천**: 사용자의 읽은 책, 평점, 선호 장르를 분석하여 추천
- **협업 필터링**: 유사한 독서 패턴을 가진 가족 구성원이 읽은 책 추천
- **콘텐츠 기반 추천**: 책의 메타데이터 (장르, 저자, 주제)를 기반한 추천

#### 3.4.2 가족 추천 (Family Curation)
- **"우리 가족이 좋아할 만한 책"**: 전체 가족의 선호도를 통합하여 추천
- **세대별 추천**: 나이대별로 맞춤형 도서 추천 (청소년 도서 vs 성인 도서)
- **주간 추천**: 매주 새로운 추천 도서 리스트 제공

#### 3.4.3 추천 배경 설명
- **추천 이유 표시**: "당신이 {책 제목}을 좋아했기 때문입니다" 같은 설명 제공
- **상세 정보**: 추천 도서의 줄거리, 평점, 리뷰 미리보기

---

### 3.5 통계 및 대시보드 (Statistics & Dashboard)

#### 3.5.1 개인별 통계
- **월별 독서량**: 월별 읽은 책의 권수 (막대 그래프)
- **장르별 분포**: 읽은 책의 장르 비율 (파이 차트)
- **독서 시간 통계**: 누적 읽기 시간
- **평점 분포**: 읽은 책의 평점 분포
- **독서 습관**: 일주일 중 자주 읽는 요일, 선호 읽기 시간대

#### 3.5.2 가족 통합 통계
- **가족 전체 독서량**: 이번 달 목표 대비 현황
- **가족 구성원별 비교**: 각 멤버별 월간 독서량 비교 (순위 시스템 도입 가능)
- **가족이 함께 읽는 책**: 여럿이 동시에 읽고 있거나 읽은 공통 책
- **가장 인기 있는 책**: 가족 내에서 가장 높은 평점을 받은 책

#### 3.5.3 대시보드 페이지
- **홈 대시보드**: 나의 진행 중인 책, 최근 리뷰, 가족 활동 한눈에 보기
- **통계 페이지**: 상세한 차트와 그래프
- **가족 비교 페이지**: 가족 전체 통계 및 멤버 간 비교

---

### 3.6 알림 및 소셜 기능 (Notifications & Social Features)

#### 3.6.1 알림
- **댓글 알림**: 내 리뷰에 댓글이 달렸을 때
- **추천 알림**: 새로운 도서 추천이 생겼을 때
- **가족 활동 알림**: 가족 구성원이 새 책을 다 읽었을 때
- **목표 달성 알림**: 월간 독서 목표 달성 축하

#### 3.6.2 소셜 기능
- **읽고 싶은 책 공유**: 특정 책을 읽고 싶다고 표시, 가족에게 공유 가능
- **도서 추천하기**: 특정 가족 멤버에게 도서 직접 추천
- **공동 읽기 챌린지**: "이번 달 함께 읽기" 같은 챌린지 (추후 확장)

---

### 3.7 어드민 및 관리 기능 (Admin & Management)

#### 3.7.1 가족 관리 (부모 역할)
- **가족 구성원 관리**: 멤버 추가/삭제/권한 변경
- **가족 정보 수정**: 가족명, 가족 소개 수정
- **가족 통계 확인**: 전체 독서 현황 모니터링

#### 3.7.2 콘텐츠 관리
- **공격적인 리뷰 신고**: 부적절한 댓글/리뷰 관리
- **도서 정보 수정**: 도서 DB의 오류가 있을 시 수정 제안

### 4.2 백엔드 (Backend)

#### 런타임 및 프레임워크
- **Node.js** (LTS 버전)
- **Next.js API Routes** (초기) → **Express.js** 또는 **Fastify** (확장 시)
  - 초기: Next.js API Routes로 빠르게 프로토타입
  - 확장: 마이크로서비스 분리 시 독립 서버로 전환 가능

#### API 아키텍처
- **REST API** (초기) → **GraphQL** (추후 고려)
  - 초기: REST API로 간단한 CRUD 구현
  - 추후: GraphQL로 유연한 데이터 쿼링

#### 데이터베이스
- **PostgreSQL** (주 DB)
  - 관계형 데이터 모델링에 최적
  - JSONB 지원으로 유연한 스키마
  - 확장 가능한 성능

- **Redis** (캐싱 & 세션)
  - 추천 알고리즘 캐싱
  - 실시간 알림 푸시
  - 세션 관리

#### ORM
- **Prisma**
  - 타입 안전한 데이터 접근
  - 자동 마이그레이션
  - 개발 생산성 향상
  - 복잡한 쿼리도 타입 안전함

#### 인증 & 인가
- **NextAuth.js v5** (Next.js 통합)
  - 이메일/비밀번호 인증
  - OAuth 제공자 통합 (Google, Apple, Naver, Kakao)
  - JWT 또는 세션 기반 인증
  - 가족 계정 관리 확장 가능

#### AI/추천 엔진
- **Python (FastAPI)**를 별도 마이크로서비스로 구현
  - 추천 알고리즘 (협업 필터링, 콘텐츠 기반)
  - 자연어 처리 (리뷰 감정 분석)
  - REST API로 Node.js와 통신

대안: **TensorFlow.js** 또는 **ONNX.js** (브라우저 내 경량 모델)

- **Google Books API**: 도서 정보 검색

- **Kakao Books API**: 국내 도서 정보

### 5.1 주요 테이블

```
테이블: users
- id (UUID, PK)
- email (VARCHAR, UNIQUE)
- password_hash (VARCHAR)
- family_id (FK → families)
- role (ENUM: parent, child, admin)
- nickname (VARCHAR)
- avatar_url (VARCHAR)
- preferred_genres (JSONB)
- monthly_reading_goal (INT)
- created_at, updated_at

테이블: families
- id (UUID, PK)
- family_name (VARCHAR)
- family_description (TEXT)
- created_by (FK → users)
- created_at, updated_at

테이블: books
- id (UUID, PK)
- isbn (VARCHAR, UNIQUE, NULLABLE)
- title (VARCHAR)
- author (VARCHAR)
- publisher (VARCHAR)
- publication_date (DATE)
- cover_image_url (VARCHAR)
- summary (TEXT)
- genres (JSONB)
- external_id (VARCHAR) - Google Books API ID 등
- created_at, updated_at

테이블: reading_logs
- id (UUID, PK)
- user_id (FK → users)
- book_id (FK → books)
- status (ENUM: to_read, reading, completed)
- started_at (DATE)
- completed_at (DATE, NULLABLE)
- rating (INT, NULLABLE) - 1~5
- pages_read (INT, NULLABLE)
- total_pages (INT, NULLABLE)
- created_at, updated_at

테이블: reviews
- id (UUID, PK)
- user_id (FK → users)
- book_id (FK → books)
- rating (INT) - 1~5
- review_text (TEXT)
- has_spoiler (BOOLEAN)
- created_at, updated_at

테이블: comments
- id (UUID, PK)
- review_id (FK → reviews)
- user_id (FK → users)
- comment_text (TEXT)
- created_at, updated_at

테이블: recommendations
- id (UUID, PK)
- recommender_id (FK → users)
- recommended_to_id (FK → users) - NULL이면 전체 추천
- book_id (FK → books)
- reason (TEXT)
- created_at
```

## 6. 보안 고려사항

### 6.1 인증 & 인가
- **이메일 인증**: 가입 시 이메일 확인
- **강력한 비밀번호 정책**: 최소 8자, 대문자/소문자/숫자/특수문자
- **JWT 토큰**: 접근 토큰 (짧은 수명), 리프레시 토큰 (긴 수명)
- **역할 기반 접근 제어 (RBAC)**: 부모/자식 권한 분리

### 6.2 데이터 보호
- **HTTPS**: 모든 통신 암호화
- **CORS**: 도메인 기반 접근 제어
- **SQL Injection 방지**: Prisma ORM 사용으로 방지
- **XSS 방지**: 리뷰/댓글 입력 sanitization
- **Rate Limiting**: API 남용 방지

### 6.3 프라이버시
- **프라이빗 커뮤니티**: 가족 멤버만 접근 가능
- **GDPR/CCPA 규정**: 데이터 삭제 권리 제공
- **개인정보 최소화**: 필요한 정보만 수집
